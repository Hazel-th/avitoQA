# BUGS

## BUG-001 — В выдаче животных показана игрушка

**Краткое описание:** среди объявлений с котятами отображается карточка `Персидский котёнок плюшевый`.

**Шаги воспроизведения:**
1. Открыть страницу выдачи, как на скриншоте.
2. Просмотреть карточки объявлений во втором ряду.

**Фактический результат:** в выдаче есть игрушка вместо живого животного.

**Ожидаемый результат:** в категории животных должны отображаться только релевантные объявления о животных.

**Приоритет:** P0  
**Окружение:** Desktop web, по приложенному скриншоту.

## BUG-API-001 — `POST /api/1/item` возвращает тело, не соответствующее контракту коллекции

**Краткое описание:** успешный ответ создания объявления не содержит созданную сущность, хотя в Postman-коллекции для `200 OK` описан объект объявления.

**Шаги воспроизведения:**
1. Отправить `POST https://qa-internship.avito.com/api/1/item` с валидным JSON-телом.
2. Посмотреть тело ответа.

**Фактический результат:** сервис возвращает JSON вида `{"status":"Сохранили объявление - <uuid>"}`.

**Ожидаемый результат:** сервис должен вернуть объект созданного объявления с полями `id`, `sellerId`, `name`, `price`, `statistics`, `createdAt`, как описано в Postman-коллекции.

**Серьёзность:** High  
**Приоритет:** P1  
**Окружение:** `https://qa-internship.avito.com`, reproduced on April 18, 2026.

## BUG-API-002 — Повторный `DELETE /api/2/item/{id}` возвращает `404`, но в теле ошибки пишет статус `500`

**Краткое описание:** после удаления объявления повторный `DELETE` возвращает HTTP `404`, однако поле `status` в JSON-теле содержит значение `"500"`.

**Шаги воспроизведения:**
1. Создать валидное объявление через `POST /api/1/item`.
2. Удалить его через `DELETE /api/2/item/{id}`.
3. Повторно выполнить `DELETE /api/2/item/{id}` для того же идентификатора.
4. Проверить HTTP-код и тело ответа.

**Фактический результат:** HTTP-ответ — `404 Not Found`, тело ответа — `{"result":{"message":"","messages":null},"status":"500"}`.

**Ожидаемый результат:** поле `status` в теле ошибки должно быть согласовано с HTTP-кодом ответа и содержать `404`.

**Приоритет:** P2  
**Окружение:** `https://qa-internship.avito.com`, reproduced on April 18, 2026.

## BUG-API-005 — `GET /api/2/statistic/{id}` для malformed UUID возвращает HTTP `404`, но в теле ошибки статус `400`

**Краткое описание:** у ручки статистики `v2` при невалидном идентификаторе HTTP-код ответа не совпадает с полем `status` в JSON-теле.

**Шаги воспроизведения:**
1. Выполнить `GET https://qa-internship.avito.com/api/2/statistic/not-a-uuid`.
2. Проверить HTTP-код и тело ответа.

**Фактический результат:** HTTP-ответ — `404 Not Found`, тело ответа содержит `{"result":{"message":"передан некорректный идентификатор объявления","messages":null},"status":"400"}`.

**Ожидаемый результат:** при malformed UUID сервис должен возвращать согласованный client error; HTTP-код и поле `status` в теле должны совпадать.

**Приоритет:** P1  
**Окружение:** `https://qa-internship.avito.com`, reproduced on April 18, 2026.

## BUG-API-006 — Для ошибок типа данных в `POST /api/1/item` возвращается misleading message `не передано тело объявления`

**Краткое описание:** при передаче поля неверного типа, например `sellerID` строкой, сервис сообщает, что тело объявления не передано, хотя тело запроса фактически присутствует.

**Шаги воспроизведения:**
1. Отправить `POST /api/1/item` с JSON-телом, где `sellerID` передан строкой, либо `price` передан строкой.
2. Проверить тело ответа.

**Фактический результат:** сервис отвечает `400`, но поле `status` содержит `не передано тело объявления`, а `result.message` остаётся пустым.

**Ожидаемый результат:** сервис должен вернуть понятную ошибку валидации типа данных и согласованный код/статус ошибки.
 
**Приоритет:** P2  
**Окружение:** `https://qa-internship.avito.com`, reproduced on April 18, 2026.

## BUG-API-007 — `POST /api/1/item` не принимает нулевые значения в `statistics`

**Краткое описание:** сервис трактует `0` в полях `likes`, `viewCount` и `contacts` как отсутствие обязательного значения.

**Шаги воспроизведения:**
1. Отправить `POST /api/1/item` с валидным телом, где `statistics = {"likes": 0, "viewCount": 0, "contacts": 0}`.
2. Проверить ответ сервиса.

**Фактический результат:** сервис отвечает `400 Bad Request` и сообщает, что поле `likes` обязательно.

**Ожидаемый результат:** нулевые значения счётчиков должны считаться валидными, а объявление должно успешно создаваться.
 
**Приоритет:** P2  
**Окружение:** `https://qa-internship.avito.com`, reproduced on April 18, 2026.
